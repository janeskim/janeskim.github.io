Classes: Bringing Together Objects and Methods


Ruby is an object-oriented programming language. This means that getting anything done in Ruby requires working with objects: objects contain data and we ask objects to manipulate and return data in useful forms.

Last week we looked into an example of the method #each_by in the Enumerable module. All the methods in the Enumerable module only work on objects that are lists. This is because different types of objects can only execute certain types of tasks. An object's ability to use a method depends on its class.

Let's take a real life example: when we have the numbers "2" and "4" we can complete addition, subtraction, multiplication and division. When we have the words "cat" and "pig", we can't run those mathematical actions on the words. "cat" - "pig" = ? Doesn't work quite the same. If we think about this in Ruby terms, the objects "2" and "4" are in the class number and the objects "cat" and "pig" are in the class words. Different types of classes have certain characteristics and those classes can execute a specific set of methods.

In Ruby, we can actually create our own classes in Ruby. We can also create objects that belong to a class and have the objects execute specific tasks. For our example, let's create a class of bikes. What are some of the characteristics a bike has? A bike has a color, wheels, and gears. What kinds of actions can a bike do? The bike can speed up, slow down and come to a full stop. Now, let's translate this into Ruby code.



class Bike

   def initialize(color, wheels, gears)
     @color = color
     @wheels = wheels
     @gears = gears
   end

   def specs
     puts "My bike is #{@color}, has wheels that are #{@wheels} tall and #{@gears} speeds."
   end

   def ride
     puts "We are riding the bike."
   end

end



This is a very simple example of our new class, Bike. Anytime we create an object that belongs to the class Bike we must specify three characteristics: the color, the wheels, and the gears. Once the object has been created, it can take the methods specs and ride.

Let's see our method in action here.



>>turbo = Bike.new("red", "29 inches", 3)
>>turbo.specs
"My bike is red, has wheels that are 29 inches tall and 3 speeds."
>>turbo.ride
"We are riding the bike."



This is a very, very simple example of how classes work. Here we created a new object called turbo, defined it as belonging to the class Bike and specified three characteristics. We called the methods specs and ride on the object turbo and executed two functions.

There are many ways to change characteristics, add methods and change our classes, but the important thing to know are every object in Ruby belongs to a class, which gives it a set characteristics and methods.
