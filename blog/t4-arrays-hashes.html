Data Structures: Arrays and Hashes


One of the basic building blocks in Ruby are variables. Just like algebra, we write Ruby variables to represent a specific value. We also call the variable when we want to use its value in the code.

>> x = 1234567890
1234567890
>> x * 2
2469135780

In the example above, the variable x represents the number 1234567890. In Ruby, variables can hold numbers, strings (words and sentences) and boolean expressions (a term in Ruby referring to true or false statements).

Arrays and hashes are a specific type of variable called a data structure. Just like variables, data structures hold value but arrays and hashes are different in that they can hold more than one value. In other words, arrays and hashes are lists. Arrays and hashes differ from each other in a few important ways, so let's examine the syntax and functionality of each item.

ARRAYS

This is how we write an array.

>> my_array = [1, 2, 3, 4, 5]

Just like the example of the variable x, we first give the array a name my_array and set it equal = to a value, which is the list contained within the brackets [1, 2, 3, 4, 5]. To note the obvious, we separate different items in the list using commas. This goes across the board for arrays and hashes. We can name our array anything we want and we can fill the array's brackets with any combination of numbers, strings and boolean expressions.

In order to access a specific value in the array, we have to know its index number. Naturally we would call 1 the first element in the array, but arrays start from the index number 0. To print 1 from our array, this is how we would write it.

>> puts my_array[0]
1

In the example above, we use the method puts to print out 1. Methods are like commands, and arrays have a plethora of methods to manipulate the data. In our very first example with the variable x, we used the method * (or multiply) to manipulate our variable. Like I said, there are SO MANY methods we can use on the data in my_array, but the important thing to know is arrays are indexed lists.

HASHES

This is how we write a hash.

>> my_hash = { "name" => "Jane", "city" => "Chicago", "ice_cream" => "always"}

We have the name for our hash my_hash and set it equal = to a set of values in the curly brackets. Values are separated by commas, just like arrays. However, unlike arrays where the value is referred to by its index number, hashes create key-value pairs. In the example for my_hash, the key "name" refers to its value pair "Jane"; the key "city" refers to its value pair "Chicago"; and the key "ice cream" refers to its value pair "always". Keys refer to their values using the symbol =>.

(I'm sure you also noticed all those quotation marks up there. We tell Ruby that we're using strings with quotation marks, but we don't need the quotation marks for numbers or booleans.)

Let's call the first key-value pair for my_hash. The code is pretty similar to arrays.

>> puts my_hash["name"]
"Jane"

A good way to remember the functionality of hashes is to think of a dictionary: the word we are looking up is the key and its definition is the value pair. There are many, many methods for accesing and using the data in hashes but the foundational take-away for hashes is they are key-value pairs.

So it looks like we've covered some of the basic data structures for Ruby. Not too difficult to understand, right?
