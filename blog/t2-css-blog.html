CSS Positioning: The Difference between Static, Relative, Absolute and Fixed Positioning


We use CSS to style our HTML elements such as changing the font on a paragraph or adding a color to the background, but we can also use CSS to place elements on a page.There are four types of CSS positioning values: static, relative, absolute and fixed.

When things are in the normal flow, they are already in static positioning. Declaring position: static is the same as not declaring at all. HTML elements are rendered from top to bottom, and child and sibling elements sit inside the parent element when things are in static positioning.

When an element in declared to be in relative positioning, it is still in the normal flow but slightly altered. In other words, the element is placed in relation to its position in the normal flow. By using the properties top, bottom, left, and right, we can move the element a certain number of pixels from the top, bottom, left or right of where it would normally be. Since the element is still part of the normal flow, it continues to work with the other HTML elements. Child elements will still be nested in their parent elements and elements will move from top to bottom, but we can specify values which will move the elements from their static position.

Unlike static and relative positioning, absolute positioning takes an element completely out of the normal flow. If it has a parent element, it is positioned in the top left of the parent element. If it does not have a parent element, it is positioned relative to the browser window. We can use the properties top, bottom, left, and right to adjust the element's location on the page.

Like the absolute element, the fixed element is also taken out of the normal flow. However, it stays fixed to the position on the browser and does not scroll with the page. I'm sure you've seen websites with a navigation bar at the top that always remains there, even when you're scrolling through the page--that's an example of a fixed element.
