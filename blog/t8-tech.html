Ruby Recursions: Recognizing Patterns and Repetition


In Ruby, a recursion is a method that calls itself. Recursions are helpful because they allow us to break a problem down into smaller, repetitive parts. A recursive method will continue to call itself and execute these mini steps until the problem is finally solved.

Recursive methods have a few important characteristics. First, they include a base case. This is the condition that must be met in order for the recursion to stop. The recursion also specifies what happens to the code to reduce it to its end goal.

Here's a simple example of a recursion:



>> def counting(number)
>>   return if number == 5
>>   puts number
>>   countdown(number + 1)
>> end

>> counting(1)
1
2
3
4
5


In our example above, our recursive method takes one argument, number. Following this is our base conditional: return if number == 5. This means that our recursion will check the number each time, and if the number equals 5, the recursion will return the number 5 and exit the method. The method's main goal is to print out the number as indicated by puts number. Lastly, after the method has checked the number and printed it out, it calls itself inside itself countdown(number + 1) -- but this time, the method changes its argument by adding 1 to the number. That's end of our recursive method. Below that, we call the method and supply it with the argument counting(1). As you can see, our recursion continues until the number equals 5.

We could have used a much simpler loop to execute this action, so recursions aren't necessarily the best solution to every problem. The greatest difference between loops such as for, while and times loops is that recursions call the method within itself. A great example of the elegance and utility of a recursive method can be seen when calculating factorials.

Factorials multiply a series of descending natural numbers.
4! = 4 * 3 * 2 * 1

To write a formula for this, factorials essentially work like this:
n! = n * (n - 1) * (n - 2 ) * (n - 3) ... until n is equal to 1.

We're subtracting 1 from the new number 'n', so another way to look at factorials is like this:
n! = n * (n - 1)!

We also have our base conditional: the recursion stops when n is equal to 1.
Break loop when (n - 1) = 0

We have two repetitions, two small recursive loops in our factorial, and we also have our base conditional. Let's translate our idea into code:



>> def factorial(number)
>>   if number == 1
>>     return 1
>>   else
>>     return number * factorial(number - 1)
>>   end
>> end

>> factorial(4)
24


We have all the ingredients to our recursion above: using a simple if/else statement, we created the base condtional that will break the loop and return 1 when the number is equal to 1. Otherwise, our recursion calls itself with a small modification to the argument return number * factorial(number - 1) so that we can reduce our recursive method to its final solution. When we call our method factorial(4), it returns 24.

Recursions are complex loops that provide a lot of elegance to a solution when recursions are applicable. Here are some great resources on recursions:
