Ruby Recursions: Recognizing Patterns and Repetition


In Ruby, a recursion is a method that calls itself. Recursions are helpful because they allow us to break a problem down into smaller, repetitive parts. The recursion will continue these steps until the problem is finally solved.

Recursive methods have a few important characteristics. First, they include a base case. This is the condition that must be met in order for the recursion to stop. The recursion also specifies what happens to the code to reduce it to its end goal.

Here's a simple example of a recursion:



 def counting(number)
   return if number == 5
   puts number
   countdown(number + 1)
 end

 counting(1)
  1
  2
  3
  4
  5


The greatest difference between loops such as for, while and times loops is that recursions call the method within itself. A great example of the elegance and utility of a recursive method can be seen when calculating factorials.

Factorials multiply a series of descending natural numbers.
4! = 4 * 3 * 2 * 1

To write a formula for this, factorials essentially work like this:
n! = n * (n - 1) * (n - 2 ) * (n - 3) ... until n is equal to 1.

We're subtracting 1 from the new number 'n', so another way to look at factorials is like this:
n! = n * (n - 1)!

We have two repetitions, two small recursive loops in our factorial. Let's translate our idea into code:



 def factorial(number)
   if number == 0
     return 1
   else
     return number * factorial(number - 1)
   end
 end

 factorial(4)
  24
