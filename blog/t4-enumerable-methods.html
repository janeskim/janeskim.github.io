Enumerables: Iterating With Finer Control


Last week we discussed how arrays and hashes are ordered lists. We touched upon methods and explained them briefly as commands that can be executed by objects. This time we're going more indepth into a related Ruby concept about methods called Enumerables.

The Enumerable is a module. A module is a collection of objects that share a common set of traits. In this case, all Enumerable share a specific structure: the object must be able to apply a method over elements within itself. For example, lists can apply a method over each item in the list. Arrays and hashes fall under the Enumerable module, and as a result, they can accept Enumerable methods. The repetition of a method over each item in an object is called iteration.

To talk more about how Enumerables iterate over an object, let's look at the Enumerable method, #group_by. Simply put, the method group_by breaks a list into groups based on specified criteria. It takes a list as its input and returns a hash as its output.

Let's at the general syntax for #group_by then go into more specific examples.

>>input.group_by { |x| x }

Aside from a few exceptions, methods are called using a . period. In our syntax example, our input on the left side of the period is the object receiving the method. On the right side of the we have the method group_by which is being sent to the object.

Next are a pair of curly brackets. This is where we expand the method group_by with more specific information on how we want it to group the input. The variable within the pipes |x| acts as a placeholder for each item in the input list. Think of the pipes as a door: each item of the input collection is lining up to walk through these pipes in order access the block of code specified in the curly brackets. In our syntax example above, we only have x after the pipes, but we can expand this part of the code to specify how we want to group each of the items.

Let's take a look at a live example.

>>input_array = [ "Jane", "Jeff", "Tonya", "Hannah", "Harry" ]
>>input_array.group_by { |name| name.length }
{ 4 => ["Jane", "Jeff"] , 5 => ["Tonya", "Harry"] , 6 => ["Hannah"] }

We called the method .length on our variable name. The method counted the number of characters in each item in the input_array. The output hash took the length of item as the key and grouped the input into arrays for the value.

Here's another example.

>>input_array = [ 2, 6, 5, 4, 10, 3, 3, 51 ]
>>input_array.group_by { |number| number.even? }
{ true => [ 2, 6, 4, 10 ] , false => [ 5, 3, 3, 51 ] }

This time we used the method with a question mark at the end even? which returns a true or false, also known as a boolean value, and grouped the data accordingly.
